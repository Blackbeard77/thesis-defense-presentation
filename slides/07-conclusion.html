Key Findings

Answering the research question — within the controlled Unity prototype:

DIALOG TREE
Most efficient Most predictable

Near-zero runtime cost and latency. Full narrative control. But manual authoring effort grows exponentially with scale.

LOCAL LLM 🏆
Viable compromise Zero runtime cost

Gemma 3 (1B) achieved 0% error rate and latency comparable to cloud — entirely offline. Surprising result.

CLOUD LLM
Most capable Recurring cost

GPT-4o was the only model that inferred branching rules from prompt alone. But ~15% of game price goes to dialog costs per player.

A locally deployed LLM can approximate deterministic dialog behavior surprisingly well using only prompting and contextualization — without fine-tuning, at zero ongoing cost, and with competitive latency.

Conclusion

Recommendations for Developers

  • Small games / tight scope?
    Use dialog trees — minimal complexity, fully predictable, easy to test.
  • Need flexibility without external deps?
    Use a local LLM with careful prompt design — near-deterministic behavior is achievable.
  • Need max capability and can afford it?
    Use cloud LLMs — but plan for recurring costs and vendor dependency.
  • Never treat LLMs as black boxes
    Always embed them in pipelines with prompt templates, context selection, and output filtering.
The bottom line

There is no universally optimal solution. The best dialog system depends on game genre, team size, budget, and desired player experience.
Plan for scale early

Whether you use a handcrafted tree, a hybrid approach, or an LLM-centric architecture — this decision strongly influences maintainability over the game's full lifetime.

Future Work

Several natural next steps emerge from this prototype:

  • Multiple NPCs & complex quests — find where performance and consistency start to break down
  • Concurrent dialog sessions — evaluate local LLM behavior under parallel load
  • Hybrid architectures — combine dialog trees with LLM fallback for best of both
  • Retrieval-Augmented Generation (RAG) — inject game lore dynamically to improve grounding
  • Purpose-trained models — fine-tune small models specifically for NPC dialog
  • User studies — formally evaluate perceived dialog quality and immersion
Revisit as models improve

Language models continue to evolve rapidly. Repeating similar prototype studies in the future will reveal how the feasibility landscape shifts as smaller models become more capable.

This thesis contributes a developer-oriented comparison framework — a starting point, not a final answer.
