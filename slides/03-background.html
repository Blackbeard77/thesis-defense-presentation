Background

NPC Dialog Systems in Video Games

NPC dialog systems mediate structured conversations between players and virtual characters — using predefined trees, state machines, or rule-based logic to control what each character says and when.

  • Early approaches relied on hand-crafted grammars — labor intensive and hard to scale
  • Believability depends on the cognitive gap between expected and perceived NPC behavior
  • Dialog must align with plot, genre, NPC identity and player actions
  • Modern ML techniques make automated NPC dialog generation increasingly feasible

Background

Dialog Trees

Dialog trees represent a finite state machine — nodes hold NPC responses, edges represent player choices. Every path is pre-authored and deterministic.

Conversation Start
↓
Hi
Who are you?
Where is everybody?
↓ (unlocks)
Taken by who?
How do I survive?
✓ Strengths

  • Full narrative control
  • Near-zero runtime cost
  • Easy to test & validate
  • No external dependencies
✗ Limitations

  • Rigid — no improvisation
  • Authoring effort grows exponentially
  • Every branch must be manually written
  • Not scalable for large games

Background

Large Language Models (LLMs)

LLMs are transformer-based neural networks trained on massive text corpora — they generate responses probabilistically at runtime based on learned statistical patterns.

System Prompt + NPC Persona
↓
Player Input (token)
↓
Transformer Self-Attention
↓
Generated NPC Response
Key difference from dialog trees

Dialog trees produce identical output for identical input. LLMs sample from a probability distribution — same prompt can yield different responses each time.
⚠ Challenges

  • Hallucinations — fluent but wrong outputs
  • Non-determinism complicates testing
  • Requires prompt engineering for consistency
  • Computational overhead at runtime
